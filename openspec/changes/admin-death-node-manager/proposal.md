# Proposal: Admin Death Node Manager

Status: proposed
Target Version: 2.4.1

## 問題

死亡節點目前可由死亡背包回收流程停用，但管理員缺少集中查看、依玩家篩選、診斷與清理節點的工具。舊資料、故障流程或測試資料可能留下無效、孤兒或重複節點，必須提供 Server-authoritative 管理介面。

## 目標

- 提供管理員死亡節點 GUI。
- 可依玩家、Dimension、狀態與時間篩選。
- 可查看節點、傳送至節點附近、停用或永久刪除節點。
- 支援單筆與批次清理。
- 所有操作具有權限驗證、確認流程與稽核紀錄。

## 非目標

- 不允許一般玩家編輯死亡節點。
- 本變更不加入玩家自行重新命名死亡節點。
- 不在 2.4.1 內建立完整死亡歷史分析系統。
- 不因刪除節點而自動刪除仍存在的死亡背包實體。

## 相容性

既有 SavedData 不需強制 migration。管理介面必須能辨識缺少部分新欄位的舊節點，並以唯讀或可清理狀態顯示。
