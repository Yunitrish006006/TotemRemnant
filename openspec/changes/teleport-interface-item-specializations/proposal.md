# Change: Teleport Interface Item Specializations

## Why

目前 Space Unit 傳送介面只接受普通羅盤。地圖、書本與回生羅盤雖然都具有導航、知識或死亡追蹤的主題，但無法參與 Nexus 傳送流程，也沒有形成不同的路線選擇。

本變更將普通羅盤、回生羅盤、書本與已繪製地圖列為可開啟傳送介面的物品，並讓每種介面提供互不重疊的特化效果。

## What Changes

- 新增傳送介面類型：
  - `COMPASS`
  - `RECOVERY_COMPASS`
  - `BOOK`
  - `FILLED_MAP`
- 四種物品都可：
  - 右鍵空氣，以玩家當下位置作為 `PLAYER` 來源開啟傳送介面。
  - 右鍵已註冊磁石，以該磁石作為 `LODESTONE` 來源開啟傳送介面。
- 普通羅盤維持現有通用行為，不提供數值加成，並保留磁石註冊、探索、管理及好友互動能力。
- 回生羅盤對玩家自己的 `DEATH` 目的地提供死亡傳送特化，降低傳送偏差。
- 書本作為「路線典籍」介面，對已探索的固定磁石目的地降低準備時間與石碑磨損率，但不降低成本或偏差。
- 已繪製地圖在目標位於該地圖的 Dimension 與覆蓋範圍內時，降低食物等價成本與傳送偏差。
- 空白地圖 `minecraft:map` 初期不列入有效介面；只有具有地圖資料的 `minecraft:filled_map` 可使用地圖特化。
- 同一趟傳送只使用一種介面特化，不疊加主手與副手中的其他介面物品。
- Client 不得提交可信加成、地圖範圍、最終成本或偏差；所有條件與公式由 Server 重新解析。

## Impact

### Affected systems

- Space Unit 物品互動與介面開啟流程。
- 傳送來源 context、Server quote 與 `TeleportSession`。
- 傳送成本、準備時間、偏差與結構磨損公式。
- `SpaceUnitMapPayload` 與地圖資訊面板。
- 翻譯、tooltip、文件與測試。

### Compatibility

- 不更改既有 Space Unit UUID、Owner、探索、好友或權限資料。
- 普通羅盤維持現有傳送數值與所有管理能力。
- 新欄位若需要持久化，必須使用 optional codec 或只保存於短期 session；舊世界不得因缺少介面欄位而無法載入。
- 現有紫水晶催化折抵與介面特化可以共同作用，但計算順序必須固定且由 Server 完成。

### Risks

- 把「可開啟傳送介面」錯誤擴大成可註冊、探索或管理磁石。
- Client 偽造介面類型、地圖 ID、地圖範圍或特化數值。
- 玩家報價後更換物品，仍保留原本介面加成。
- 動態 `PLAYER` 目標移出地圖範圍後仍使用舊折扣。
- 地圖範圍檢查洩漏好友精確位置。
- 整數捨入使成本、偏差或磨損降到不合理的 0。
