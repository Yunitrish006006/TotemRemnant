# OpenSpec 13.7 persistence evidence

Status: complete.

## Entity NBT round-trip

Existing `CopperGolemPersistenceGameTest` verifies mode, running state, revision, gathering tool durability, gathering storage Components, area and manual rules through Entity NBT serialization and reload.

## Real chunk unload and reload

`CopperGolemChunkPersistenceGameTest` places the Copper Golem, Home and target in a remote chunk, removes the force ticket, waits for the server to confirm the chunk is unloaded, reloads it, and resolves the entity by its original UUID.

The test verifies Home, target, fuel, activity, tool and storage Components, manual rules and scan cursor after reload.

## Three independent Dedicated Server JVMs

`CopperGolemRestartProbe` and the `runCopperRestartProbe` Loom configuration execute `seed`, `recover` and `verify` against the same normal Dedicated Server world.

The probe verifies two persisted state generations, including:

- entity UUID;
- mode, stopped state, revision and activity;
- fuel stack and fuel ticks;
- Home binding and Copper Chest inventory;
- gathering area and target;
- tool durability and Components;
- storage quantity and Components;
- manual rules and scan cursor;
- target block changes.

GitHub Actions runs this probe after the existing death-backpack restart probe and requires an `.ok` marker from every phase.
