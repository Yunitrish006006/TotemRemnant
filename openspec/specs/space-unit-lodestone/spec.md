# Totem Nexus：Space Unit 磁石傳送規格

## 1. 目標

以世界中的磁石與傳送石碑作為固定空間錨點，保留 Blossom 的 Space Unit 概念，但改為 Server authoritative、世界 SavedData、探索後解鎖及地圖式介面。

系統包含固定磁石、羅盤、石碑穩定度、傳送偏差、食物成本、跨維度紫水晶、好友、人體磁石、死亡節點與分散重生。

## 2. Space Unit

類型：

```java
LODESTONE
PLAYER
DEATH
TEMPORARY
SYSTEM
```

固定節點至少保存 UUID、名稱、Dimension、磁石位置、Owner、可見性、管理員、允許玩家、狀態、結構快照與時間戳。顯示名稱不得作為主鍵；相同 `Dimension + BlockPos` 只能有一個有效固定節點。

固定磁石方塊被破壞、替換，或在已載入區塊中不再是磁石時，對應 `LODESTONE` Space Unit 必須停用並從位置索引移除；同座標重新放置磁石視為新的節點，不得沿用舊節點的 Owner、權限或私人狀態。

可見性：

```java
PRIVATE
FRIENDS
PUBLIC
HIDDEN
```

玩家必須同時「已探索」且「有權限」，節點才會出現在傳送地圖。公開或好友可見不代表自動公開未探索座標。

Owner／Administrator 可在目標磁石附近切換固定節點的私人／好友／公開可見性；`FRIENDS` 可見性只對已互加好友且已探索該節點的玩家生效。

Owner 可在目標磁石附近將線上玩家加入或移出 Administrator 名單。Owner／Administrator 可在目標磁石附近將線上玩家加入或移出允許名單。名單調整必須由 Server 驗證來源、目標磁石、互動距離、區塊載入、目標方塊與操作者權限；Owner 本身不得被加入或移出 Administrator／允許名單。

## 3. 磁石與羅盤互動

### 右鍵磁石

玩家手持羅盤右鍵磁石：

- 未註冊：第一次右鍵只掃描石碑並開啟預覽 GUI；同一玩家需在確認期限內按下確認，或再次右鍵同一顆磁石，Server 才建立 Space Unit 並綁定羅盤。預設確認期限為 30 秒。
- 已註冊：綁定羅盤、開啟地圖或管理節點。
- 開啟地圖時，來源必須是玩家這次實際右鍵的磁石，而不是羅盤先前綁定的磁石。
- 只有 Owner／Administrator 可改名、改可見性、管理權限及重新校準。
- 改名必須由 Server 驗證 Owner／Administrator、已探索、玩家位於目標磁石附近、目標區塊已載入，且目標方塊仍是磁石。
- 改可見性必須由 Server 驗證 Owner／Administrator、已探索、玩家位於目標磁石附近、目標區塊已載入，且目標方塊仍是磁石。
- 調整 Administrator／允許名單必須由 Server 驗證 Owner／Administrator、已探索、玩家位於目標磁石附近、目標區塊已載入，且目標方塊仍是磁石；Administrator 名單僅 Owner 可調整。
- 註冊確認 GUI 只能送出 pending 註冊確認；Server 必須重新驗證 pending 的 Dimension／座標、玩家仍在磁石附近、目標方塊仍是磁石、玩家仍手持羅盤，並拒絕過期或偽造確認。

### 左鍵啟用

玩家必須親自抵達，手持羅盤左鍵已註冊磁石，Server 才把該節點加入玩家探索資料。

Server 必須驗證玩家互動距離、實際擊中方塊、Dimension 與 Space Unit。Client 不可只傳 UUID 遠端啟用。

### 羅盤操作

- 右鍵磁石：註冊、綁定、以該磁石作為來源開啟地圖，或管理節點。
- 右鍵空氣：以玩家自身當下位置作為臨時 `PLAYER` 來源錨點開啟地圖。
- 右鍵玩家：開啟好友／玩家傳送選單。

羅盤只保存 Space Unit UUID 與資料版本；真實位置由 Server 查詢。磁石來源地圖與後續傳送請求必須帶有 `sourceType=lodestone` 與來源 Space Unit UUID，Server 重新驗證玩家仍在該磁石附近。玩家自身來源必須帶有 `sourceType=player`，Server 使用玩家當下位置重新建立臨時來源，不信任 Client 座標。預設磁石來源啟動半徑為 8 格。

## 4. 傳送石碑與穩定度

結構快照包含：

- 完整度。
- 對稱性。
- 能量或共振程度。
- 空間干擾。
- 環境穩定性。
- 磨損程度。
- 結構等級。

石碑材料及退化規則必須使用 Tag 或資料驅動設定。

掃描只在首次註冊、附近方塊改變、手動校準、傳送前驗證、損壞後及世界載入後首次使用時執行，禁止每 tick 完整掃描。

手動校準必須由 Server 驗證 Owner／Administrator 權限、玩家已探索該節點、玩家位於目標磁石附近、目標區塊已載入，且目標方塊仍是磁石後才可重新掃描。Client 只能提交來源型態、來源 UUID 與目標 UUID，不得提交穩定度或結構快照。

路線穩定度由來源穩定度、目標穩定度、距離、Dimension 與目標類型共同決定。固定磁石到固定磁石最穩定；死亡殘響與人體磁石有額外懲罰。

## 5. 傳送成本

不得消耗經驗值。

同維度傳送優先消耗：

1. 飽和度。
2. 飢餓值。
3. 不足部分由食物物品支付。

Server 依 Food Component 計算食物價值。自動選取不得預設消耗特殊效果食物、帶自訂資料的食物或玩家保護的物品。

跨 Dimension 必須額外消耗紫水晶碎片，且來源與目標石碑均達到最低結構等級。

Server 報價需包含：飢餓成本、食物需求、紫水晶、準備時間、穩定度、偏差及結構損壞率。正式開始前必須重新計算。

## 6. 傳送 Session

傳送具有準備時間，使用 Server tick session，不得使用裸 Thread。

取消條件：

- 玩家移動過遠。
- 玩家受傷、死亡、離線或切換 Dimension。
- 玩家不再持有啟動羅盤。
- 來源或目標失效。
- 成本不足。
- 玩家目標撤回同意。
- Server 關閉功能。

## 7. 偏差與安全落點

低穩定度造成水平、垂直及朝向偏差。人體磁石與死亡節點有額外偏差。

建議級距：

| 穩定度 | 偏差 |
|---|---:|
| 95%–100% | 0–1 格 |
| 80%–95% | 1–3 格 |
| 60%–80% | 3–8 格 |
| 40%–60% | 8–20 格 |
| 20%–40% | 20–48 格 |
| 0%–20% | 高機率失敗 |

最終落點由 Server 搜尋，必須有安全支撐、碰撞箱淨空，且不在岩漿、火焰、粉雪、虛空、世界邊界外或禁止區域。

## 8. 結構磨損

傳送可能損壞來源與目標石碑。損壞率受距離、跨維度、穩定度、既有磨損與結構等級影響。

優先使用方塊退化：

```text
石磚 → 裂石磚
深板岩磚 → 裂深板岩磚
銅方塊／切製銅 → 下一氧化階段
```

損壞只能作用於結構掃描器已確認的石碑方塊，不得破壞附近無關建築。損壞後重新掃描並更新磨損與節點狀態。

## 9. 地圖式 GUI

地圖以來源錨點為中心：右鍵磁石時來源是該磁石，右鍵空氣時來源是玩家自身當下位置。同 Dimension 節點按相對 X/Z 顯示；其他 Dimension 使用分頁或圖層。

只能同步玩家已探索且有權限查看的節點。不得同步未探索磁石、無權限節點、離線玩家最後位置或 Hidden 節點。線上好友作為 `PLAYER` 目標顯示時，Client payload 只能收到粗略化後的顯示座標與距離，不得收到玩家精確座標。

支援拖曳、縮放、Dimension 切換、搜尋、分類、好友篩選、收藏及節點資訊面板。資訊面板顯示距離、穩定度、成本、時間、偏差、損壞率、可見性、結構階級及目前玩家是否可管理。成本與風險摘要應以圖示搭配數字呈現，並提供 hover 說明。

可管理的固定磁石資訊面板應提供基礎名單調整入口。初期名單調整只支援線上玩家名稱；離線玩家查詢、完整名單瀏覽與批次管理可由後續完整管理介面補上。

好友篩選必須依 Server payload 明確標記，只顯示由雙向好友擁有且已通過探索與權限過濾的節點；不得由 Client 只依 `FRIENDS` 可見性字串推測。

## 10. 好友與人體磁石

玩家持羅盤右鍵玩家，發送好友邀請；若對方已先邀請自己，則接受邀請並建立雙向好友關係。

地圖 GUI 必須提供好友管理入口。好友管理介面至少顯示既有好友、待接受邀請、已送出邀請與對方在線狀態；玩家可從介面移除好友或取消任一方向的待處理邀請。移除好友後，兩人之間的 `PLAYER` 目標可見性必須立即失效，既有未完成的人體磁石傳送同意也必須取消。

好友只能看到自己已探索且可見性為 `FRIENDS` 或 `PUBLIC` 的節點。

線上好友可作為 `PLAYER` Space Unit：

- 僅限雙向好友，且每次傳送都需要對方確認。
- 玩家在地圖選擇線上好友後只會送出傳送請求；目標玩家必須在期限內手持羅盤右鍵請求者，Server 才能開始傳送 session。
- 精確位置只由 Server 取得；地圖 payload 必須粗略化座標與距離，避免 Client 取得即時精確位置。
- 成本、時間、偏差與石碑壓力高於固定節點。
- 倒數完成時使用目標最新位置。

## 11. 死亡節點

死亡背包成功建立後，建立只對死亡玩家可見的 `DEATH` Space Unit。死亡事件視為已探索，不必再次到場。

死亡節點成本與偏差高於固定磁石；跨維度仍需紫水晶。死亡背包回收後節點不可再傳送並從預設地圖隱藏。

## 12. 分散重生

世界規則：

```text
deadrecall:dead_recall_distributed_spawning
```

Boolean，建議預設 `false`。

開啟後，沒有有效床、重生錨或既有個人分散重生點的玩家，獲得一個持久個人重生點。優先順序：床 → 重生錨 → 個人分散重生點 → 世界出生點。

不得每次死亡重新抽選。候選位置使用有限隨機採樣，依安全、地形、生態域、距離及玩家密度評分。玩家越多的區域，被選中機率越低。

個人分散重生點不得寫入玩家的原版床／重生錨資料；Server 只在原版 `respawnConfig` 不存在且 gamerule 開啟時，用 DeadRecall SavedData 的個人點建立臨時重生轉移。若既有個人點變得不安全，Server 可在附近修正到安全表面；若無法修正，才重新抽選。

建議密度分數：

```text
nearby512 × 4
+ nearby2048 × 2
+ nearby8192
```

候選點與其他個人重生點保持可配置最小距離，建議 1,000–2,000 格。分散重生點不會自動成為 Space Unit。

## 13. 世界資料

建議 SavedData：

```text
DeadRecallSpaceUnitSavedData
DeadRecallSpaceDiscoverySavedData
DeadRecallFriendSavedData
DeadRecallDistributedSpawnSavedData
```

所有資料必須有 `dataVersion` 與 migration。

## 14. Networking

禁止沿用 `String operation + 完整 SpaceUnit`。每個操作使用獨立 Payload。

Client 不得提交可信座標、Dimension、成本、穩定度、偏差、安全落點、損壞結果、玩家密度或重生位置。

地圖重新整理與傳送請求可提交來源型態與 UUID，但 `sourceType=lodestone` 必須由 Server 查世界 SavedData 及實際方塊位置，`sourceType=player` 必須由 Server 使用請求玩家當下位置。Client 不得以任何 Payload 指定來源座標。

## 15. 驗收條件

- 玩家必須到場並持羅盤左鍵磁石才能啟用。
- 未註冊磁石首次右鍵不得立即建立節點；必須完成同一玩家、同一位置的 GUI 或二次右鍵確認。
- 右鍵磁石負責註冊、確認、綁定與管理。
- 右鍵磁石開啟地圖時，來源必須是被點擊的磁石。
- 右鍵空氣開啟地圖時，來源必須是玩家自身當下位置。
- 磁石方塊被破壞或替換後，舊節點不再顯示、不可傳送，且同座標重放磁石必須重新註冊。
- 未探索或無權限節點不顯示。
- 傳送不消耗經驗值。
- 同維度成本使用飢餓值與食物。
- 跨維度必須使用紫水晶。
- 傳送具備倒數、取消、偏差與安全落點。
- Owner／Administrator 可在目標磁石附近從地圖資訊面板手動校準，Server 重新掃描石碑並刷新報價。
- Owner／Administrator 可在目標磁石附近從地圖資訊面板重新命名固定磁石節點。
- Owner／Administrator 可在目標磁石附近從地圖資訊面板切換私人／好友／公開可見性；未探索玩家仍不得看到公開或好友節點。
- Owner 可在目標磁石附近從地圖資訊面板調整 Administrator 名單；Owner／Administrator 可調整允許名單；偽造封包不得越權調整名單。
- 結構損壞只影響已辨識石碑方塊。
- 好友邀請由羅盤右鍵玩家觸發。
- 地圖 GUI 可開啟好友管理介面，並列出好友、待接受邀請、已送出邀請與在線狀態。
- 從好友管理介面移除好友或取消邀請後，Server SavedData 必須更新，相關人體磁石目標與未完成同意必須立即失效。
- 好友節點可見性需同時滿足雙向好友與已探索。
- 地圖好友篩選只顯示由雙向好友擁有且已探索、有權限查看的節點。
- 人體磁石只允許線上雙向好友，且每次傳送都必須由目標玩家用羅盤右鍵請求者確認。
- 人體磁石不洩漏精確位置；地圖 payload 不得同步玩家精確座標。
- Gamerule 關閉時維持原版重生；開啟時仍優先使用床與重生錨。
- 分散重生點必須持久保存，不得每次死亡重新抽選，且不會自動成為 Space Unit。
- 所有永久資料在 Server 重啟後存在。
- 偽造封包不能建立節點、啟用磁石或任意傳送。
