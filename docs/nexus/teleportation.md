# Space Unit 傳送

Space Unit 傳送採用 Server-authoritative session。Client 只提交來源型態、來源 UUID 與目標 UUID；Server 重新驗證權限、探索、位置、成本、結構快照與安全落點。

## 好友玩家目標

- 線上雙向好友可以作為 `PLAYER` 傳送目標。
- 雙向好友關係本身視為持續授權；選取好友後直接建立傳送倒數，不需要目標玩家再用羅盤確認。
- 非好友、只有單向邀請、離線玩家與玩家自己都不能作為有效 `PLAYER` 目標。
- 只有 session 實際建立成功且目標 UUID 確認為玩家端點後，目標玩家才會收到 `Teleport: requester → target` 通知。
- 倒數期間與完成前都會重新解析目標；解除好友會立即取消任一方向的相關 PLAYER 目標 session，目標離線、死亡或被移除時也不會完成傳送。
- PLAYER 目的地失效時會明確區分「玩家已離線」、「玩家已死亡或不可用」與「好友關係已解除」。固定磁石、死亡節點等非玩家目的地仍使用一般目的地失效訊息。
- 完成時使用目標玩家的最新 Server 位置，Client 地圖仍只收到粗略座標與距離。
- 舊版逐次確認流程已從核心程式移除；羅盤右鍵玩家只處理好友邀請與接受。

## 成本

### 同維度

- 基礎食物成本依距離計算，約每 384 格增加 1 點。
- 成本優先從飽和度扣除，再扣飢餓值。
- 飢餓值至少保留 1 點。
- 若仍不足，從玩家物品欄消耗安全的食物補足。
- 基礎食物成本最高限制為 20 點。

### 跨維度

- 使用較高食物成本。
- 原始紫水晶碎片成本至少為 2，路線越不穩定成本越高。
- 固定磁石石碑中的紫水晶催化方塊可以降低最終成本。
- 來源與目標固定石碑的催化方塊數量合併計算；每 4 個有效催化方塊折抵 1 個紫水晶碎片。
- 折抵後的跨維度最終成本最低為 1，不會完全免費。
- `PLAYER`、`DEATH` 與其他沒有固定石碑的端點不提供該端折抵。
- 跨維度來源與目標磁石都必須至少達到 Tier 1。

初版有效催化方塊為完整紫水晶方塊 `minecraft:amethyst_block`，類型由 `deadrecall:space_unit_amethyst_catalysts` block tag 定義，可由資料包擴充。

| 來源與目標催化方塊總數 | 紫水晶折抵 |
| ---: | ---: |
| 0–3 | 0 |
| 4–7 | 1 |
| 8–11 | 2 |
| 12–15 | 3 |

例如原始成本為 4，兩端共有 8 個有效紫水晶方塊時，最終消耗為 2 個紫水晶碎片。

目前地圖沿用既有欄位顯示折抵後的最終成本；原始成本、催化數量與折抵明細會在後續 Payload／UI 階段分開顯示。

創造模式玩家不消耗傳送資源。

## 石碑結構與穩定度

磁石不是單獨放置就會取得完整穩定度。Server 會掃描磁石周圍 `5×3×5` 區域中的有效結構方塊，計算完整度、對稱度、共鳴度、Tier 與紫水晶催化方塊數。

目前有效結構方塊包括石磚系列、深板岩磚／磚瓦系列、紫水晶方塊，以及銅方塊、切製銅方塊與雕紋銅方塊系列。

| 結構條件 | Tier |
| --- | ---: |
| 少於 8 個有效結構方塊 | 0 |
| 至少 8 個 | 1 |
| 至少 24 個 | 2 |

建議至少在磁石同一高度對稱放置一圈 8 個有效方塊：

```text
S S S
S L S
S S S
```

`L` 是磁石，`S` 是有效結構方塊。

## 定向重新掃描

為避免 SavedData 保存舊的結構快照，系統只在必要時重新掃描相關節點：

- 開啟磁石來源地圖時，重新掃描來源磁石。
- 在地圖選取目標時，重新掃描來源與該目標，然後由 Server 回傳新報價。
- 按下傳送時，再次重新掃描來源與目標並重新驗證。
- 倒數完成前重新解析兩端並計算最終成本，避免玩家在報價後拆除催化方塊仍取得舊折抵。

不會因為刷新地圖就重掃所有 Space Unit，因此掃描成本不會隨節點總數線性增加。即使 Client 還顯示舊的「不可傳送」報價，傳送按鈕也會交由 Server 使用最新結構重新判定。

## 安全食物

系統只計算可安全消耗的普通食物。帶有 Custom Data 的物品不會被自動當作傳送燃料，降低誤吃特殊物品的風險。

## 準備階段

按下傳送後會建立倒數 session。準備時間由距離、Dimension、路線穩定度與目標類型共同決定，目前限制在 40 至 300 ticks。

下列情況會取消傳送：

- 玩家死亡、被移除或離線。
- 玩家受到實際傷害。
- 玩家不再手持羅盤。
- 玩家切換 Dimension。
- 玩家離開啟動位置超過 4 格。
- 來源或目標在倒數期間失效。
- 成本或權限條件在倒數期間不再成立。
- `PLAYER` 目標離線、死亡、被移除，或雙方不再是好友。
- 任一方在好友管理 GUI 解除雙方好友／邀請關係；相關 session 會立即移除。

成本會在實際完成傳送前再次計算，只有成功找到安全落點後才扣除。

## 安全落點

Server 會在目標附近搜尋可站立位置：

- 玩家腳部與頭部必須為空氣且沒有流體。
- 地板必須可支撐玩家。
- 必須位於世界邊界和有效高度內。
- 排除岩漿、火、營火、仙人掌、岩漿塊與細雪等危險方塊。

磁石節點以磁石上方作為基準；其他節點使用自己的記錄位置。

## 路線穩定度

路線穩定度受下列因素影響：

- 來源與目標本身的共鳴度。
- 是否跨維度；跨維度會套用額外衰減。
- 同維度距離。
- 來源是否為玩家臨時錨點。
- 目標是否為死亡、玩家或臨時節點。

穩定度低於 0.2 時會阻止傳送。
