# 磁石註冊、探索與羅盤操作

## 註冊磁石

手持羅盤右鍵尚未註冊的磁石時，伺服器會建立一個 `LODESTONE` Space Unit：

- 記錄 Dimension、BlockPos、Owner、可見性、狀態與結構快照。
- 預設為私有節點。
- 將該節點 UUID 與資料版本寫入羅盤 Custom Data。
- 同時寫入原版 `LODESTONE_TRACKER`，保留羅盤指向效果。

若手上羅盤堆疊大於一個，生存模式會拆出一個獨立綁定羅盤，避免整疊羅盤共用同一份節點資料。

## 結構快照

磁石註冊時會掃描周圍 `5×3×5` 區域，記錄：

- 有效結構方塊數量與完整度。
- 結構對稱度。
- 共鳴度 `resonance`。
- 石碑 Tier。
- 後續磨損與環境穩定度欄位。

裸磁石通常只有 Tier 0 與偏低共鳴度。至少對稱放置 8 個有效結構方塊，可使石碑達到 Tier 1；24 個以上可達到 Tier 2。

結構改動後不需要重新建立節點。開啟地圖、選取傳送目標或按下傳送時，Server 會只重新掃描本次使用的來源與目標磁石，更新 SavedData 中的快照。

## 探索磁石

玩家必須親自到達磁石，手持羅盤左鍵磁石，才會把該節點加入自己的探索資料。

- 未註冊磁石會提示先註冊。
- 無查看權限的節點不能探索。
- 重複探索只會顯示已探索提示。
- 探索記錄按玩家 UUID 永久保存。

## 右鍵已註冊磁石

- 有權限時重新綁定羅盤。
- 尚未探索時，只提示玩家先左鍵探索。
- 已探索時，以**這次實際右鍵的磁石**作為地圖來源。
- 不會使用羅盤先前綁定的另一個磁石作為本次來源。
- 開啟地圖前會重新掃描這個來源磁石的結構。

## 右鍵空氣

手持羅盤右鍵空氣時，會以玩家當下位置建立臨時 `PLAYER` 來源錨點並開啟地圖。

Server 只接受 `sourceType=player` 與玩家自己的 UUID，來源 Dimension 與座標會直接使用玩家當下狀態重新建立，Client 不能提交座標。

## 權限與可見性

目前已完成私有 Owner 權限基礎。地圖資料同時套用：

1. 玩家是否已探索該節點。
2. 玩家是否有權查看該節點。
3. 節點是否仍為 ACTIVE。

未探索、無權限、Hidden 或失效節點不應同步到 Client。
