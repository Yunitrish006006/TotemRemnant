# Space Unit 地圖

Space Unit 地圖是自訂 Client GUI，由 Server 傳送經探索與權限過濾後的節點資料。

## 來源錨點

地圖有兩種來源：

| 來源型態 | 開啟方式 | Server 驗證 |
| --- | --- | --- |
| `lodestone` | 手持普通羅盤、回生羅盤、書本或已繪製地圖，右鍵已探索磁石 | 查詢 SavedData、確認磁石仍存在、玩家有權限且在 8 格內 |
| `player` | 手持上述四種介面物品右鍵空氣 | 使用請求玩家的 UUID、當下 Dimension 與 BlockPos |

來源型態與 UUID會包含在重新整理及傳送請求中，但 Client 不得提交來源座標。

只有普通羅盤能註冊或探索磁石、管理磁石及操作好友關係。回生羅盤、書本與已繪製地圖只提供地圖與傳送入口；空白地圖不是有效介面。Server 另保存短期介面 context，驗證確切使用手、物品類型與已繪製地圖的 map ID，Client 不能自行指定這些身分。

## 顯示內容

- 以來源錨點為中心顯示同維度節點的相對 X／Z 位置。
- 不同 Dimension 以分頁切換。
- 左側節點清單可滾動。
- 地圖支援縮放與節點選取。
- 選取節點後顯示 Server 計算的傳送資訊。
- 標題列顯示目前介面物品圖示與名稱；選取目標後顯示介面加成是否有效及條件。
- 非普通羅盤介面隱藏好友管理、磁石校準、重新命名、可見性與存取權限操作。

## 傳送報價資訊

每個地圖項目可包含：

- 路線穩定度。
- 節點結構等級。
- 同維度距離。
- 基準與最終食物等價成本，以及最終成本如何分配至飽和度、飢餓值與物品欄食物。
- 額外食物需求與玩家可用食物點數。
- 跨維度紫水晶原始成本、來源／目標石碑催化數、折抵、最終成本與現有數量。
- 基準與最終準備時間。
- 基準與最終最大水平偏差。
- 抵達傷害機率報價。
- 基準與最終的固定石碑磨損機率。
- 介面類型、特化是否有效及 Server 選定的說明訊息。
- 是否允許傳送及阻擋原因。

偏差、抵達傷害與石碑磨損都由 Server 最終報價實際執行；書本只降低獨立的石碑磨損率，不會連帶降低抵達傷害。

資訊面板的食物、時間、偏差與磨損圖示會顯示 Server 提供的 `base → final`；沒有變化時只顯示單一數值。食物 tooltip 另顯示這次實際從飽和度、飢餓值與物品欄配置的點數。創造模式的基準、最終與實際配置皆為 0。

已繪製地圖的覆蓋判斷完全在 Server 進行：map ID 只用來查詢原版 `MapItemSavedData`，Dimension、中心、比例尺與目標精確位置不接受 Client 輸入。好友 `PLAYER` 項目仍只傳送 64 格網格粗略座標、64 格級距距離與加成 active／inactive 狀態，不會因地圖特化洩漏即時精確位置。

## 同步限制

單次 Payload 最多包含 128 個節點。Server 只建立玩家可見且已探索的節點列表，不把完整世界節點資料交給 Client；decoder 會拒絕未知介面 enum、過長說明 key、負數／超量節點、不合法的時間／偏差／機率、不一致的食物配置、`final` 高於 `base`，以及不可能的紫水晶報價欄位。
