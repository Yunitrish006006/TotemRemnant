# Space Unit 地圖

Space Unit 地圖是自訂 Client GUI，由 Server 傳送經探索與權限過濾後的節點資料。

## 來源錨點

地圖有兩種來源：

| 來源型態 | 開啟方式 | Server 驗證 |
| --- | --- | --- |
| `lodestone` | 手持羅盤右鍵已探索磁石 | 查詢 SavedData、確認磁石仍存在、玩家有權限且在 8 格內 |
| `player` | 手持羅盤右鍵空氣 | 使用請求玩家的 UUID、當下 Dimension 與 BlockPos |

來源型態與 UUID會包含在重新整理及傳送請求中，但 Client 不得提交來源座標。

## 顯示內容

- 以來源錨點為中心顯示同維度節點的相對 X／Z 位置。
- 不同 Dimension 以分頁切換。
- 左側節點清單可滾動。
- 地圖支援縮放與節點選取。
- 選取節點後顯示 Server 計算的傳送資訊。

## 傳送報價資訊

每個地圖項目可包含：

- 路線穩定度。
- 節點結構等級。
- 同維度距離。
- 飽和度與飢餓值成本。
- 額外食物需求與玩家可用食物點數。
- 跨維度紫水晶碎片成本與現有數量。
- 準備時間。
- 最大水平偏差。
- 抵達傷害機率報價。
- 是否允許傳送及阻擋原因。

目前偏差與傷害資訊已進入報價與 GUI，但實際隨機偏差、抵達傷害及石碑磨損仍列為後續功能。

## 同步限制

單次 Payload 有節點數量上限。Server 只建立玩家可見且已探索的節點列表，不把完整世界節點資料交給 Client。
