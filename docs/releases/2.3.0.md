# DeadRecall 2.3.0 變更清單

發佈日期：2026-07-13

## 重點

- 新增 Totem Nexus / Space Unit 第一批可玩流程：磁石註冊、探索、地圖 GUI、傳送報價與傳送 session。
- Discord Bridge 擴充為完整事件轉播與伺服器狀態路由，支援 Cloudflare Worker 多頻道發送。
- 銅魁儡管理 GUI 改為容器介面，燃料、採集工具、採集倉庫與玩家背包使用原版拖曳／Shift-click 流程；所有物品欄位使用原版風格且對齊 hitbox 的 slot 底板，面板會依 GUI 可用寬高伸縮並重排。
- 銅魁儡目前動作改以狀態圖示顯示，hover tooltip 顯示 activity 文字，避免 header 文字遮住控制項。
- 銅板手新增 tooltip，說明綁定狀態、Shift+右鍵開啟 GUI、來源/Home、目標設定與原版 slot 操作。
- 死亡背包整合死亡節點，讓死亡地點可以進入 Space Unit 地圖流程。
- 補齊多人伺服器需要的權限、好友、分散重生與管理稽核基礎。

## Nexus / Space Unit

- 新增 Space Unit 世界 SavedData、探索資料、好友資料與分散重生資料。
- 手持羅盤右鍵磁石可註冊／綁定 Space Unit；未註冊磁石會先顯示石碑預覽 GUI，確認後才建立節點。
- 手持羅盤左鍵已註冊磁石可探索節點；未探索或無權限節點不會同步到客戶端。
- 右鍵磁石開地圖時，以玩家實際點擊的磁石作為來源錨點。
- 右鍵空氣開地圖時，以玩家自身當下位置作為臨時來源錨點。
- 新增 Space Unit 地圖 GUI：維度切換、搜尋、類型篩選、好友共享篩選、排序、收藏與節點資訊面板。
- 地圖資訊面板以圖示顯示距離、飽和度、飢餓值、食物、紫水晶、時間、穩定度、偏差與磨損率。
- 新增 server-side 傳送報價：食物成本、紫水晶成本、準備時間、穩定度、偏差與結構磨損率。
- 新增傳送 session：倒數、移動取消、受傷取消、離線取消、成本重新驗證、安全落點與抵達風險傷害。
- 同維度傳送消耗飽和度、飢餓值與安全食物；飢餓值最低保留 1，不再把飢餓度小於 1 的玩家強制設成 1。
- 跨維度傳送需要紫水晶碎片與最低石碑階級。
- 新增石碑結構掃描、穩定度、階級、磨損與傳送後方塊退化。
- 石碑退化規則 data pack 化，並支援 Nether Bricks、Polished Blackstone Bricks、Copper Grates、Copper Bulbs 等材料。
- 固定磁石被破壞或替換後會停用對應 Space Unit；同座標重放磁石不會沿用舊 Owner、權限或私人狀態。
- 地圖可管理磁石支援重新校準、重新命名、私人／好友／公開可見性切換。
- Owner 可調整 Administrator；Owner／Administrator 可調整允許名單。
- 新增好友流程：羅盤右鍵玩家送出邀請；對方右鍵回來可接受並建立雙向好友。
- 新增好友管理 GUI：顯示好友、待接受邀請、已送出邀請與在線狀態，可移除好友或取消邀請。
- 線上好友可作為 `PLAYER` 目的地；地圖只顯示粗略座標與距離，每次傳送都需要目標玩家用羅盤確認。
- 死亡背包建立後會建立 `DEATH` 節點；死亡背包回收後會停用並從地圖隱藏。
- 新增分散重生 gamerule 與持久個人重生點，優先順序仍保留原版床與重生錨。

## Discord Bridge

- Cloudflare Worker 支援 `/api/mc/chat` 與 `/api/mc/server/status`。
- 支援事件格式化：玩家聊天、死亡、首次加入、加入、離開、進度、管理稽核、健康告警、死亡背包、Space Unit 公開事件、Boss 擊敗、Raid、難度、Gamerule、村民升級。
- 加入 Bot Token 頻道路由：遊戲內設定頻道 ID 且 Worker 有 `DISCORD_BOT_TOKEN` 時，優先送到指定 Discord 頻道。
- 保留 Webhook fallback：未設定 Bot Token 或頻道 ID 無效時，回退使用 `DISCORD_WEBHOOK_URLS`。
- Worker 會清理重複或無效頻道 ID，並限制單次請求最多轉送 10 個頻道。
- 加強 Secret 安全：API 回應與 log 不回傳完整 Webhook URL、Webhook token 或授權標頭。
- Discord Bridge 設定 GUI 改為只允許 OP／單人世界擁有者讀取設定，且不再把既有 API Key 同步回客戶端；API Key 欄位留空會保留目前設定。
- Discord Bridge 頻道管理改為 Server authoritative，同步最新清單並驗證 Discord snowflake ID 與最多 10 個頻道限制。
- 伺服器狀態與健康告警改為非同步轉送，避免阻塞 server tick。
- 補上 Discord Bridge OpenSpec、部署文件與 troubleshooting。

## 修正

- 修正 server 啟動時載入 client mixin config 的問題：client mixin 現在只在 client 環境載入。
- 修正 Space Unit 傳送成本中飢餓度低於 1 時的處理，不再把玩家飢餓度補成 1。
- 修正 Space Unit 目的地被破壞後仍保留舊私人綁定狀態的問題。
- 修正 Space Unit GUI 節點列表高度，讓可容納三列時正確顯示三個傳送點。
- 修正死亡背包資料安全與物品回收流程的保護基礎。
- 修正銅傀儡死亡或永久移除時，自訂燃料槽、採集工具與採集倉庫的單次掉落與清理流程。
- 修正銅魁儡管理 GUI 中燃料與採集物品只能靠按鈕移動的限制，改由原版 slot transaction 保護游標堆疊與背包同步。
- 移除銅魁儡 GUI 舊的燃料／工具／採集倉庫 serverbound payload，不再支援從主手或背包自動搜尋物品放入欄位。

## 管理員注意事項

- 伺服器與客戶端都需要更新到 2.3.0。
- 使用 Discord Bridge 的伺服器需要重新部署 `deploy/worker.js`。
- Worker secrets 建議確認：
  - `MC_API_KEY`
  - `DISCORD_WEBHOOK_URLS`
  - `DISCORD_BOT_TOKEN`，若要用頻道 ID 路由則需要
- Space Unit 新增多個世界 SavedData；升級前建議備份世界。
- Space Unit 石碑退化規則位於 `data/deadrecall/deadrecall/space_unit_degradation/vanilla.json`，可由資料包覆寫或擴充。
- 銅魁儡資料採 lazy migration；舊銅魁儡首次讀取時會補上 `deadrecall_data_version = 2`、預設 `SORTING` 模式與 revision。詳見 `docs/copper-golem/migration.md`。
