# DeadRecall 2.4.1 變更清單

發佈日期：2026-07-15

## 版本定位

2.4.1 是 DeadRecall 2.x 的資料安全與功能收尾版本。此版本優先穩定死亡背包、死亡節點、世界保存與多人伺服器基礎。第三方 player-owned inventory 整合只有在具備可重現 Minecraft 26.2 runtime、transaction rollback 與 Components 測試後才納入；目前已完成公開 addon SPI 與 Trinkets Updated 4.1.x optional compatibility。

## Discord 繁體中文事件

- Advancement Discord 通知不再於 Dedicated Server 上提前解析為英文，改以 Server-only `DiscordLocalizationService` 處理未解析 title Component。
- `task`、`goal`、`challenge` 分別顯示為「進度」、「目標」、「挑戰」。
- 村民升級訊息加入繁中通用名稱、職業及階級，例如「村民（圖書管理員）升級：新手 → 學徒」。
- 村民自訂名稱、玩家名稱及 nested literal Component 維持原樣。
- Minecraft 26.2 Dedicated Server 不提供完整 Client `zh_tw.json`，因此使用版本鎖定的 Discord 專用 immutable 翻譯子集。
- 未知 translation key 使用中文安全 fallback，不把 raw key 傳到 Discord，也不建立重複事件。
- Worker endpoint、event ID、payload 欄位、Bot Token／Webhook fallback 及多頻道路由不變。
- 死亡訊息保留原始 Component 到 formatter，中文化 Vanilla death template 與攻擊實體，同時保留玩家及 custom item literal 名稱。
- 終界龍／凋零預設名稱、自訂 Boss 名稱、raid 結果及 difficulty 顯示已接入相同 localization service。
- Missing-key warning 依 key 去重且有 128 種上限，未知內容仍使用 deterministic fallback。
- 真實本機 Worker HTTP 503 回歸確認 transport 失敗不會回拋到 Minecraft 事件。
- Server Data 資料包可透過 `data/deadrecall/deadrecall/discord_zh_tw/*.json` 覆寫指定翻譯 key；啟動及標準 `/reload` 會原子替換 immutable snapshot，移除覆寫後回到 bundled fallback。

## 遊戲配方與講台

- 覆寫 Vanilla `minecraft:lectern` recipe ID，替代配方為 4 個任意 `#minecraft:wooden_slabs`＋1 本書，產出 1 個講台。
- 不再保留以書櫃為材料的舊講台配方。
- 實際 RecipeManager GameTest 已驗證橡木、竹、緋紅蕈木、扭曲蕈木及混合木種半磚都可製作。
- 講台原版功能維持不變：可放置與取回多頁書本、開啟閱讀 Menu、翻頁更新 Comparator 訊號並產生兩 tick 紅石脈衝。
- 未就業村民可透過 Vanilla POI／Brain 流程自然認領講台並成為圖書管理員。

## 混凝土粉末掉落物硬化

- 世界中的 16 色原版混凝土粉末掉落物實際接觸水源或流動水時，整疊轉換成同色混凝土。
- 轉換使用同一個 ItemEntity，保留數量、自訂名稱等合法 Components、位置、速度、pickup delay 與 age。
- 雨水、隔著方塊的鄰近水、物品欄與容器內物品不會觸發；原版放置方塊硬化行為不變。
- 非粉末物品先經常數時間映射短路；512 個不可合併 ItemEntity 的 Server GameTest 在 10 tick 內完成，未使用世界 entity 掃描。

## Space Unit 紫水晶催化

- 固定磁石石碑中的紫水晶方塊會降低跨維度傳送成本；來源與目標每合計四個催化方塊折抵一個碎片，最終成本最低為一。
- 玩家與死亡節點不提供催化折抵；傳送開始及完成前都會重新掃描固定石碑，拆除方塊後不會沿用舊報價。
- 地圖資訊面板顯示原始成本、兩端催化數、折抵與最終成本，並使用具長度、範圍及報價一致性檢查的 Server Payload。
- 實際跨維度 GameTests 已覆蓋兩端折抵、非磁石端點、報價後拆除與 exactly-once 扣款；三次 JVM restart probe 已覆蓋缺少新欄位的舊世界遷移。

## Space Unit 傳送介面 Phase A–D

- 普通羅盤、回生羅盤、書本與具有 map ID 的已繪製地圖皆可由玩家位置或已探索磁石開啟傳送介面。
- 普通羅盤維持未註冊磁石註冊、探索、好友互動與磁石管理的獨占能力；其他介面會以明確訊息拒絕這些操作。
- Server-only context 與傳送 session 保存介面類型、確切使用手及 map ID；物品換手、更換或地圖 ID 改變時會在扣款前取消。
- 回生羅盤對玩家自己的死亡節點將最大偏差降低 50%；其他目標與其他報價欄位維持基準。
- 書本對已探索且可見的固定磁石降低 20% 準備時間與 25% 固定石碑磨損率，不改變食物、紫水晶、偏差或抵達傷害。
- 地圖 UI 顯示介面物品、加成 active／條件，以及食物、準備時間、偏差與石碑磨損的 `base → final` 明細；食物 tooltip 另顯示實際資源配置，非普通羅盤隱藏好友及磁石管理入口。
- 已繪製地圖由 Server map ID 解析 Dimension、中心與比例尺；覆蓋目標時將食物成本與最大偏差各降低 20%，不影響紫水晶、準備時間、抵達傷害或石碑磨損。
- 動態好友移出地圖範圍或 map ID／Server 地圖資料失效時，已啟用加成的 session 會在扣款前取消；Client 仍只收到好友粗略位置與加成狀態。
- Client／Server codec 會拒絕最終值高於基準值及食物配置不一致的報價；Phase D 沒有變更 SavedData schema。

## 死亡背包與死亡節點

- 死亡物品改在 `Player.dropEquipment` 入口、原版／addon equipment drops 與 `Inventory.dropAll()` 前直接封裝，不再依賴死亡後掃描附近 ItemEntity。
- 支援 Inventory、Equipment、副手、active menu 游標、玩家 2×2 合成格，以及原版工作站會返還玩家的暫存輸入格。
- Crafting Table、Anvil、Smithing Table、Grindstone、Stonecutter、Loom、Cartography Table 與 Enchanting Table 已使用明確 class／slot-range 白名單。
- Result preview、外部箱子與其他持久 block/entity inventory 不會被誤收進死亡背包。
- DeadRecall 一般背包、死亡背包與其他禁止巢狀的可攜式容器不會被放入另一個死亡背包。
- `keepInventory`、消失詛咒、環境死亡與交易失敗 rollback 已納入自動回歸。
- 舊的死亡點半徑掃描、附近 ItemEntity UUID 差集、雙重 server task 與相容 Mixin 已完整移除。
- 死亡背包清空時依背包綁定的 Death Node UUID 停用正確節點，不要求回收者必須是原死亡玩家。
- 原玩家離線時，其他玩家仍可安全回收死亡背包；玩家或 Discord 通知失敗不會阻止節點停用及空背包移除。

## 第三方物品欄相容

- 新增公開 `DeathBackpackAddonInventoryProvider`／`DeathBackpackAddonSlot` Server-side transaction SPI。
- Addon 可只提供依自身規則應死亡掉落的 player-owned slots，不需要 DeadRecall 反射掃描未知 inventory。
- Addon slots 與原版 Inventory／Equipment／transient inputs 共用同一個死亡背包 transaction。
- Commit 前會重新比對 Item、count 與完整 Data Components；來源已改變時拒絕提交。
- 部分 addon slots 已清除後若後續提交失敗，會反向原位 rollback；原位失效時回填玩家 Inventory，交給原版死亡掉落。
- 單一 provider snapshot 例外只隔離該 provider，不阻止原版 Inventory 的死亡背包擷取。
- Addon slot 中的 DeadRecall 背包或其他禁止巢狀容器不會被擷取，也不會被 DeadRecall 擅自清除。
- 內建 Trinkets Updated 4.1.x／Minecraft 26.2 optional adapter，只擷取 Trinkets 最終判定為 `DROP` 的 slots。
- Trinkets 的 `KEEP`／`DESTROY`、slot callback、enchantment 與 keepInventory 規則仍由 Trinkets 處理。
- 未安裝 Trinkets Updated 時不載入 adapter class，DeadRecall 維持可獨立啟動。
- Accessories 目前沒有 Minecraft 26.2 的直接編譯版本；Accessories compatibility layer 或其他 addon 可使用公開 SPI 接入。

## Dedicated Server 持久化驗證

- 新增 server-only `runRestartProbe`，測試程式位於 gametest source set，不會進入正式模組 JAR。
- CI 使用同一個世界目錄啟動三個獨立 Dedicated Server JVM：
  1. 建立並保存 ACTIVE Death Node、discovery 與綁定的死亡背包實體。
  2. 重新載入後，以相同 UUID 的 replacement player 回收背包並停用節點。
  3. 再次重新載入，確認節點維持 `DISABLED`、discovery 存在，且已移除的背包實體不會復活。
- 驗證涵蓋正常 world、entity region 與 SavedData 的跨程序保存／重載，不只是在記憶體內做 codec round-trip。

## 自動測試與發佈流程

- Java 25 Gradle build 與 Fabric Server GameTests 維持為必要檢查。
- Discord localization JUnit 驗證 advancement、frame type、nested Component、fallback、村民職業／階級與 exactly-once payload。
- Dedicated Server GameTests 驗證 bundled zh-TW snapshot 可在無 Client language class 的 runtime 載入。
- GameTest runtime 載入 Trinkets Updated 4.1.x 與 Yumi，驗證真實 player `DROP` slot、source 清空、Components 保存與 exactly-once 世界結果。
- 通用 addon SPI fixture 驗證成功提交、部分清槽失敗 rollback、provider 例外隔離與 portable-container 排除。
- 測試模組明確註冊全部 34 個 GameTest 類別，不再只執行早期的部分 entrypoints；新增 Hopper Minecart、Dispenser、同 tick 多玩家容器巢狀 fixture。
- 容器自動化矩陣驗證 Hopper Minecart chain、Dropper transfer、Dispenser 原生 ejection 與 Shulker sided guard 的拒絕物 exactly-once，以及普通物品 control。
- 新增 provider registry JUnit，驗證 deterministic order、immutable snapshot 與 duplicate ID 拒絕。
- 新增講台 Gameplay GameTests，驗證 runtime recipe matching、BlockEntity／Menu、書本 Components、Comparator、紅石脈衝及圖書管理員 POI。
- 新增混凝土粉末 512 ItemEntity 壓力 GameTest，驗證支援粉末原地轉換、非粉末快速退出與 entity identity 保存。
- 新增紫水晶催化跨維度傳送 GameTests、Payload codec JUnit 與舊 `space_units.dat` 三階段重啟遷移探針。
- 新增傳送介面 Phase A GameTests，驗證四種入口、空白／異常地圖拒絕、原版方塊互動優先序、普通羅盤能力邊界及 session 物品重新驗證。
- 新增 Phase B 報價矩陣 JUnit、Payload codec 範圍測試與正式 session GameTest，驗證回生羅盤／書本特化、捨入、最低值及抵達傷害與石碑磨損分流。
- 新增 Phase C 地圖中心／邊界／比例尺／Dimension JUnit 與四個 Server GameTests，驗證真實 map data、食物／偏差捨入、跨 Dimension 紫水晶不變、好友位置隱私、動態目標移出及 map ID 移除後零扣款取消。
- 新增 Phase D Payload codec 與兩個 Server GameTests，驗證書本及回生羅盤從正式報價路徑產生完整 base／final 明細；完整 suite 共 29 個 required GameTests。
- 新增 release metadata CI 檢查：`mod_version` 必須有對應的 `docs/releases/<version>.md`，且 Release Notes 索引必須包含該版本。
- Advancement、村民、系統事件與 reload GameTest 已正式註冊至 `fabric-gametest` entrypoint；snapshot 併發 JUnit 與真實 Dedicated Server reload 驗證單筆 Component 不混用新舊翻譯，且 reload 保留 bundled fallback。

## 管理員注意事項

- 伺服器與客戶端都應更新到 2.4.1。
- Discord 系統事件的顯示語系固定為繁體中文，不跟隨玩家 Client 語系。
- Trinkets Updated 不是 DeadRecall 必要依賴；未使用 Trinkets 的伺服器不需要額外安裝。
- 需要擷取 Trinkets 飾品槽時，伺服器必須安裝相容的 Trinkets Updated 4.1.x 與其必要依賴；DeadRecall 會自動啟用 adapter。
- 升級前建議備份世界，尤其是已使用 Space Unit、死亡節點或銅傀儡自訂庫存的伺服器。
- 本版沒有更改 mod ID、package、SavedData key、物品 ID、方塊 ID 或 Data Component identifier。

## 本版刻意不納入

- 離線玩家身體系統。
- Totem Core 正式拆分與 identifier migration。
- 尚未完成真人多人回歸的 Nexus 進階流程。
- 尚無 Minecraft 26.2 正式依賴版本的特定 Accessories adapter；其他 addon 先使用公開 SPI。
