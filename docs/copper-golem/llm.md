# 銅魁儡 LLM 設定

銅魁儡可使用 OpenAI-compatible Chat Completions API 協助判斷分類目的地或採集目標。LLM 是可選功能；未啟用時仍可使用手動分類與手動採集目標。

## 共用連線設定

每隻銅魁儡可設定：

- API URL
- API Key
- Model
- 測試連線

測試連線會送出最小請求，用來確認端點、認證與模型名稱可用。

## 分類模式

- 每個目的地容器可獨立啟用 LLM。
- 每個容器有自己的分類 Prompt。
- LLM 判斷結果會寫入該容器的接受或拒絕快取。
- 快取可在 GUI 中手動調整，不必再次呼叫 API。

送出判斷時應包含物品 ID、名稱、tags、Data Components 摘要及目的地 Prompt。不得把 API Key 顯示在一般玩家可見文字或伺服器 log 中。

## 採集模式

- 採集模式有獨立開關與採集 Prompt。
- 手動採集目標優先於 LLM 判斷。
- 若手動目標為空，但 LLM 設定完整，系統可依方塊 ID、名稱、tags、預期掉落與工具摘要判斷是否採集。

## 失敗處理

API 未設定、逾時、回應格式錯誤或連線失敗時，不應阻塞伺服器 tick。該次判斷應安全失敗並保留既有手動規則。
