# 銅魁儡故障排除

## 銅魁儡沒有開始工作

依序確認：

1. GUI 是否為「運作」。
2. 是否已綁定來源銅箱。
3. 是否有燃料。
4. 分類模式是否至少有一個目的地。
5. 採集模式是否已設定兩個角點、工具，以及手動目標或可用的 LLM。
6. 銅魁儡、來源與目標是否位於同一維度。

## 無法切換模式

切換前必須停止銅魁儡，並清空以下狀態：

- 銅魁儡手上的分類物品。
- 採集工具欄。
- 採集倉庫。

## 容器綁定後消失

目標方塊被破壞或不再是容器時，綁定會自動移除。只有目標 chunk 已載入時才會確認失效，避免因未載入誤刪。

## 採集目標找不到

- 確認方塊在 Corner A 與 Corner B 形成的工作區內。
- 確認目標 chunk 已載入。
- 確認方塊不是容器、流體、來源銅箱或不可破壞方塊。
- 確認工具適用且採集倉庫能接收掉落物。
- 路徑失敗的目標可能暫時被跳過。

## LLM 沒有反應

- 使用 GUI 的測試連線確認 URL、API Key 與 Model。
- 確認目的地或採集模式的 LLM 開關已啟用。
- 確認 Prompt 不為空。
- 檢查伺服器 log，但不要把完整 API Key 貼到 issue。

## 跨維度綁定失敗

這是預期行為。銅魁儡、來源銅箱、目的地與採集工作區必須在同一維度。
